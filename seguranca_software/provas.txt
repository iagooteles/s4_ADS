Um desenvolvedor está analisando a superfície de ataque de uma aplicação web e percebe que o sistema aceita diversos tipos de entradas do usuário sem a devida sanitização. Ele decide implementar o uso de expressões regulares para validar os dados antes do processamento.

Sobre a validação de entradas de dados e superfície de ataque, assinale a opção correta:

    A superfície de ataque refere-se aos pontos onde um invasor pode tentar inserir ou extrair dados, sendo a validação de entradas uma defesa proativa crítica.



Qual das seguintes etapas da metodologia de avaliação de risco é a primeira a ser realizada?

    Identificação de ativos.


É sabido que por meio do SDLC (Ciclo de Vida de Desenvolvimento de Software) é possível produzir um software de alta qualidade sem que isso implique mais custo ou tempo de produção. Assinale a alternativa que apresenta corretamente os objetivos da fase de Planejamento do SDLC.

    Definir os requisitos do software, o cronograma do projeto e o orçamento.


Sobre o ciclo de desenvolvimento de software em específico a fase de análise de requisitos, quais as principais categorias de requisitos de segurança?

    Requisitos de segurança funcional; Requisitos de segurança não funcional; Requisitos de Segurança derivada.



Em um cenário de auditoria, descobriu-se que uma aplicação utiliza variáveis de ambiente para armazenar credenciais de banco de dados de forma insegura e não validava o nome de arquivos carregados pelos usuários, permitindo a leitura de arquivos sensíveis do sistema operacional.

No que diz respeito ao tratamento de variáveis e arquivos, assinale a alternativa correta:

    Variáveis de ambiente, descritores de arquivos e nomes de arquivos são fontes de entrada que devem ser validadas para prevenir ataques de manipulação de sistema.



Qual dos itens abaixo são práticas da função de negócio de “Governança” do OpenSAMM?


    Estratégia e Métricas; Conformidade política; Educação e orientação.



Para criar um modelo de ataque é necessário considerar de forma explícita os ataques ou tipos de ataque conhecidos. Sabendo disso, assinale a alternativa que apresenta as ações realizadas durante o processo de criação de um modelo de ataque:


    Identificar os riscos que podem afetar esses ativos.

    Identificar os ativos que precisam ser protegidos.

    Identificar as vulnerabilidades que podem ser exploradas para realizar um ataque.

    Identificar as consequências de um ataque bem-sucedido.

        Todas as opções anteriores.


O ciclo de desenvolvimento de software seguro inclui diversas etapas, desde a concepção e design até a implementação e manutenção. Cada etapa deve incorporar práticas de segurança para minimizar a introdução de vulnerabilidades. Qual é a melhor prática durante a fase de design do ciclo de desenvolvimento de software para aumentar a segurança?

    Realizar uma análise de ameaças para identificar e mitigar possíveis vulnerabilidades.
